# TRTC Room Builder · 场景决策树与页面模板

> 本文件定义 room 类产品的场景分类和每种场景对应的 UI 结构模板。
> 当用户描述需求时，按决策树判断场景，然后使用对应模板生成页面。

---

## 一、场景决策树

根据用户描述的关键特征判断场景类型：

```
用户需求
  │
  ├── 多人同时视频/语音 + 屏幕共享 + 成员管理
  │   └── → meeting（视频会议）
  │
  ├── 一个主持人 + 多人语音连麦 + 无视频/仅主持人视频
  │   └── → voice-room（语音房间）
  │
  ├── 1个主播 + 大量观众 + 连麦互动
  │   └── → live-stream（直播连麦）
  │
  ├── 两人视频/语音通话
  │   └── → one-on-one（1v1 通话）
  │
  ├── 在线教育：老师讲课 + 学生举手提问
  │   └── → classroom（在线课堂）
  │
  └── 不确定 / 描述模糊
      └── → 询问用户，给出以上选项让他们选择
```

### 场景关键词映射

| 场景 | 触发关键词 |
|---|---|
| meeting | 视频会议、多人会议、远程会议、在线会议、team meeting、conference |
| voice-room | 语音房间、语聊房、开黑房、语音聊天室、电台、播客房 |
| live-stream | 直播、直播间、连麦直播、秀场直播、带货直播、live |
| one-on-one | 1v1、一对一、视频通话、语音通话、私聊通话 |
| classroom | 在线课堂、远程教学、网课、互动课堂、小班课、大班课 |

## 二、场景模板

### Template: meeting（视频会议）

**页面结构**：
```
┌──────────────────────────────────────────────────────┐
│  topbar: 换肤 | 布局 | 网络 | 会议名+计时 | 语言+用户  │
├──────────────────────────────────────────┬───────────┤
│                                          │ side-panel│
│              stage                       │ - 成员列表 │
│          (视频宫格区)                     │ - 聊天面板 │
│                                          │           │
├──────────────────────────────────────────┴───────────┤
│  bottombar: 麦克风 | 视频 | 共享 | 录制 | 成员 | 聊天 | 结束 │
└──────────────────────────────────────────────────────┘
         + modal: 设置(音频/视频/共享等) | 邀请
```

**使用的组件**（全部 60 个均可能使用）：
- topbar + toolbar-item + popover（布局选择、网络状态、房间信息）
- stage（视频区，支持九宫格/右侧列表/顶部列表布局）
- bottombar + icon-button（麦/摄/共享/录制/全屏/邀请/成员/聊天/举手/更多/设置）
- side-panel（成员列表 + 聊天面板，左右切换）
- modal（设置弹窗带侧栏导航 + 邀请弹窗）
- popover（设备选择、更多菜单、共享选项）

**交互逻辑**：
- 麦克风/视频按钮切换 `.is-off` 状态（红色+斜杠）
- 成员/聊天按钮打开侧栏（`.is-panel-open` + `.is-active`）
- 设备选择 popover 从底栏向上弹出
- 设置/邀请打开 modal

**参考实现**：`uikit/assets/themes/meeting-classic/index.html`

### Template: voice-room（语音房间）

**页面结构**：
```
┌──────────────────────────────────────────────────────┐
│  topbar: 房间名 | 在线人数 | 用户信息                    │
├──────────────────────────────────────────────────────┤
│                                                      │
│    ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                  │
│    │头像  │ │头像  │ │头像  │ │头像  │   ← 麦位格       │
│    │用户名│ │用户名│ │用户名│ │空位  │                  │
│    └─────┘ └─────┘ └─────┘ └─────┘                  │
│    ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                  │
│    │头像  │ │头像  │ │头像  │ │头像  │                  │
│    └─────┘ └─────┘ └─────┘ └─────┘                  │
│                                                      │
├──────────────────────────────────────────────────────┤
│  聊天消息滚动区域                                      │
├──────────────────────────────────────────────────────┤
│  bottombar: 麦克风 | 举手 | 消息输入 | 更多 | 离开       │
└──────────────────────────────────────────────────────┘
```

**使用的核心组件**：
- topbar（简化版：房间名 + 人数）
- avatar（麦位网格，大号头像）
- badge（说话中指示、静音标志）
- dot（在线/离线状态）
- icon-button（麦克风、举手）
- chat-message（滚动消息区）
- chat-input（底部消息输入）
- btn--end（离开按钮）

**特殊处理**：
- 无 stage 组件（语音房不需要视频区）
- 麦位网格用自定义 Grid 布局（4x2 或 3x3），每格是 avatar + 用户名 + 状态
- 主持人位视觉突出（更大/有特殊边框）
- 空麦位显示虚线 + "空位" 提示

## 三、通用构建规则

### 所有场景必须遵守

1. **HTML 根元素**必须设置 `data-theme="mc"`
2. **CSS 引入顺序**：tokens.css → layout.css → atoms → molecules → organisms
3. **所有视觉属性**通过 L2 token 引用（参见 `uikit/references/token-contract.md`）
4. **运行时状态**通过 `.is-*` class 切换
5. **组件 HTML 结构**严格遵循 `uikit/references/component-catalog.md`
6. **最小宽度 1200px**：已由 `layout.css` 内置，浏览器窄于 1200px 时出现横向滚动条，内容不压缩

### 自适应原则

- 如果用户没有指定某些功能区域（如聊天、设置），可以省略对应组件
- 如果用户描述了模板未包含的功能，可以用现有组件组合实现
- 不存在的组件不要发明，用现有组件的变体或组合替代

### 交互 JS 规范

所有交互代码放在页面底部的 `<script>` 标签中，遵循以下模式：
- popover 开关：`classList.toggle('is-open')`
- 侧栏开关：`mc-app` 切换 `.is-panel-open` + side-panel 切换 `.is-open`
- 麦/摄切换：`classList.toggle('is-off')` + 更新 label 文字
- modal 开关：`classList.toggle('is-open')`
- 点击外部关闭：document 级 click 监听
